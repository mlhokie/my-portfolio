This portfolio serves two purposes. First, it allows me to expand the scope of my technical skills and gain valuable knowledge that's applicable for a career in IT. Second, it allows viewers to learn more about me beyond my resume and the architecture used to create it.

This is a serverless portfolio hosted in the cloud utilizing AWS. Directly below is a flow chart of the architecture design. Visitors go to my domain managed by Route 53. Route 53 directs the request to CloudFront which globally distributes the site. CloudFront retrieves my portfolio from S3. The portfolio that reaches the user is made up of HTML, CSS, Javascript, and images. React is used for interactivity and Babel ensures it runs on multiple browsers. Together that makes up the frontend.

So how does the content get to S3? The source code lives in GitHub. Once the code is pushed, AWS Codebuild takes over and runs unit test with Jest, then packages the code with NPM, Webpack, and Babel where the results are stored in S3. CodePipeline then runs a Lambda function that deploys the code to the portfolio's S3 bucket. A Lambda function also sends an SNS message to notify me that the code was deployed. Together all these services ensure the site is efficient, easily scalable, and cost-effective to host.
